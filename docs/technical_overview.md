# 看了么技术说明

本文档用于承接 README 中不适合展开的产品、技术和开发约束信息。README 保持简洁，具体功能边界、页面结构、权限策略和后续开发注意事项统一放在这里。

## 产品定位

看了么是一款 Android 本地媒体整理 App，核心目标是帮助用户更快、更安全地整理手机里的照片和视频。

它的定位不是传统系统相册、云相册、社交图片 App 或 AI 修图工具，而是：

```text
本地优先的照片 / 视频整理工作台
```

核心关键词：

```text
照片整理、视频整理、滑动筛选、GIF 动图、实况照片、动态照片、收藏、回收站、当年今日、相似图片检测、成就陈列柜、排除文件夹、原生相册联动、大图缩放、视频预览、平板适配、深色模式、预测式返回、可视化自定义
```

## 产品目标

### 快速整理

用户可以在专门的整理页面中快速判断当前媒体：

- 是否保留。
- 是否收藏。
- 是否放入 App 内部回收站。
- 是否跳过。
- 当前文件夹是否应该排除在整理队列之外。

### 安全防误删

整理类 App 必须优先避免误删。推荐流程是：

```text
媒体文件
→ 用户整理判断
→ 加入 App 内部回收站
→ 用户二次确认
→ 恢复 / 彻底删除
```

默认不应直接永久删除系统媒体。彻底删除公共媒体时，应走 Android 系统授权流程，并提供明确确认。

### 本地优先

App 应主要基于本机媒体库工作：

- 不强制登录。
- 不做云同步。
- 不上传用户照片或视频。
- 不依赖服务器完成核心整理流程。
- 不加入会员、VIP、支付、广告或远程授权校验。

### 原生 Android 能力优先

开发时优先使用 Android 官方能力：

- MediaStore 扫描图片和视频。
- Storage Access Framework 选择文件夹。
- 系统 Intent 打开指定图片或视频。
- Android 预测式返回。
- Material 3。
- Jetpack Compose。
- Room 和 DataStore。

## 功能体系

### v2.2.9 功能状态

v2.2.9 是一次围绕照片整理体验、动态照片支持、相似图片检测、成就系统与发布优化的综合更新：

- GIF 动图可在整理页、收藏、回收站、时间线和大图预览中直接播放。
- 实况 / 动态照片支持识别 Android Motion Photo、常见动态照片线索和同名伴随视频段。
- 照片整理页当前卡片懒加载实况信息，当前卡片可自动播放一次，后续支持长按再次播放。
- 首页、照片整理页、视频整理页显示预计可释放空间，提示色使用主题色。
- 当年今日与系统回收站改为长方形照片墙，减少不必要的文件信息展示。
- 相似图片检测支持后台检测、进度显示、继续检测、指纹缓存复用和多特征候选匹配。
- 成就系统升级为本地成就陈列柜，展示完成率、XP、稀有度、分类筛选与解锁状态。
- Release 构建启用 R8 压缩与资源裁剪，APK 体积从约 54 MB 降至约 6.1 MB。

首页是整个 App 的总入口，建议包含：

- 开始整理按钮。
- 图片整理入口。
- 视频整理入口。
- 收藏入口。
- 回收站入口。

- 服务与支持内容应和设置页统一。

### 图片整理

图片整理是核心模块之一，主要功能包括：

- 读取本地图片。
- 生成图片整理队列。
- 图片预览。
- 滑动整理。
- 保留 / 跳过。
- 加入回收站。
- 收藏。
- 重新随机。
- 排除文件夹过滤。
- 点击放大查看。
- GIF 动图播放。
- 实况 / 动态照片识别与预览。
- 双指缩放查看细节。
- 拖拽查看局部。
- 调用系统相册打开当前图片。
- 图片切换渐变。
- 误触保护。
- 当前卡片懒加载实况解析，避免影响首次进入速度。
- 底部栏统一展示媒体信息，避免照片上重复信息层遮挡内容。

重点验收项：

- 进入整理页面时，不应先显示“暂无队列”再突然显示图片。
- 点击图片放大后，应能缩放查看细节。
- 调用系统相册时，应打开当前图片，而不是相册首页。
- 手势判定不能太模糊，避免轻微滑动触发回收站操作。
- 图片切换时背景不能突然闪白。
- 授权图库后应能通过轻量媒体扫描稳定读取照片 / 视频。
- 实况照片播放不应循环过度影响整理节奏。
- 长按实况提示应减少遮挡，优先使用边框挂件样式。
- 重新随机按钮应有效，且不能破坏顶部布局或当前页面结构。
- 加入回收站应是可恢复状态，不应直接永久删除。

### 视频整理

视频整理与图片整理类似，但需要额外关注播放器生命周期。主要功能包括：

- 扫描本地视频。
- 生成视频整理队列。
- 视频缩略图展示。
- 视频预览播放。
- 播放 / 暂停控制。
- 加入回收站。
- 收藏。
- 跳过 / 保留。
- 重新随机。
- 排除文件夹过滤。
- 调用系统视频查看器。
- 平板宽屏适配。
- 快速切换时稳定释放播放器资源。

推荐逻辑：

```text
进入视频整理页
→ 加载当前视频 URI
→ 创建 / 复用 ExoPlayer
→ 播放或预览
→ 切换下一个视频
→ 释放旧资源
→ 加载新视频
```

重点验收项：

- 视频整理页面不能因播放器生命周期问题闪退。
- 快速切换视频时应释放旧资源。
- 视频加载失败时应有兜底 UI。
- 视频页面应适配平板宽度。

### GIF 与动态照片

GIF 与动态照片是照片预览能力的一部分，不应拆成独立云端处理流程：

- GIF 使用本地解码播放，在整理页、收藏、回收站、当年今日和大图预览中复用。
- 动态照片识别基于 MediaStore 元数据、MIME、文件名、路径、轻量头尾片段和同目录同名短视频伴随文件。
- 实况解析应在当前卡片懒加载，避免阻塞首次进入照片整理页。
- 当前卡片可自动播放一次实况，长按可作为再次播放入口。
- 大图预览和收藏预览应复用同一套动态照片播放能力。
- 无法识别或播放动态片段时，应回退为普通照片预览。

### 系统相册联动

用户点击“用相册查看”后，应直接打开当前图片或当前视频，而不是打开相册首页。

实现要点：

- 使用当前媒体的 `content://` URI。
- 设置正确 MIME 类型。
- 添加 URI 读取权限。
- 处理 Android 版本差异。
- 处理部分系统相册不支持直接定位的情况。
- 打不开时提供兜底提示。

推荐 Intent 思路：

```kotlin
Intent(Intent.ACTION_VIEW).apply {
    setDataAndType(uri, mimeType)
    addFlags(Intent.FLAG_GRANT_READ_URI_PERMISSION)
}
```

### 收藏

收藏用于保存用户认为重要、不想误删、想之后再看的照片或视频。

功能包括：

- 收藏图片。
- 收藏视频。
- 收藏列表。
- 取消收藏。
- 点击放大查看。
- 视频播放。
- 系统相册打开。
- 和回收站状态同步。

注意事项：

- 收藏页不能只是列表，应能直接预览。
- 收藏中的媒体如果被加入回收站，应明确展示状态。
- 收藏和回收站状态不能互相冲突。
- 收藏页应复用统一媒体预览组件。

### 回收站

回收站是安全机制。

功能包括：

- 查看已加入回收站的图片。
- 查看已加入回收站的视频。
- 放大查看图片。
- 播放视频。
- 恢复媒体。
- 彻底删除。
- 批量删除。
- 清空回收站。
- 删除前确认。
- 预计可释放空间、照片 / 视频数量和 30 天恢复提示。
- 长方形照片墙布局，减少不必要的文件信息展示。

重点：

- 加入回收站只是 App 内部标记，不等于系统永久删除。
- 彻底删除需要二次确认。
- 对 Android 公共媒体的删除需要走系统授权流程。
- 回收站内部也必须支持点击放大查看。

### 当年今日

当年今日是情绪价值功能，用于帮用户重新发现以前的照片和视频。

功能包括：

- 根据当前日期匹配历史照片 / 视频。
- 展示往年同一天拍摄的媒体。
- 点击查看。
- 收藏。
- 加入回收站。
- 系统相册打开。
- 按年份分组。
- 长方形照片墙布局，优先展示照片本身，减少文件信息干扰。

### 相似图片检测

相似图片检测在本机完成，不能上传用户照片：

- 支持后台检测与进度显示。
- 支持继续检测和指纹缓存复用，避免大相册重复计算。
- 多特征检测包括 pHash、dHash、aHash、颜色特征与候选桶匹配。
- 使用候选桶降低大相册检测压力，并减少误判概率。
- 检测结果写入本地指纹、桶和相似分组表。

### 成就系统

成就系统是本地统计展示，不依赖账号或远程授权：

- 展示本地成就陈列柜。
- 支持完成率、XP、稀有度、分类筛选与解锁状态。
- 进度来自本地整理记录、收藏、回收站、释放空间和连续整理数据。

### 排除文件夹

排除文件夹是整理 App 的关键能力。

功能包括：

- 图片排除文件夹。
- 视频排除文件夹。
- 设置页管理排除文件夹。
- 整理页直接新增排除文件夹。
- 调用 Android 原生文件夹选择器。
- 支持移除排除规则。
- 扫描媒体时自动跳过被排除目录。

适合排除的目录：

- 微信缓存。
- QQ 缓存。
- 截图目录。
- 下载目录。
- 表情包目录。
- 工作素材目录。
- 其他不想整理的文件夹。

推荐使用：

```text
Storage Access Framework

```

## 核心交互

### 滑动整理

整理页支持滑动判断，但必须避免误触：

- 轻微滑动只移动卡片。
- 超过阈值才触发动作。
- 动作触发前显示方向提示。
- 加入回收站时有视觉反馈。
- 操作后可以撤销。
- 关键删除不直接永久执行。

### 点击放大和缩放查看

所有展示图片的地方都应支持点击放大：

- 整理页。
- 收藏页。
- 回收站。
- 当年今日。

放大后支持：

- 双指缩放。
- 拖拽移动。
- 双击放大。
- 返回关闭。
- 系统相册打开。

### 动画

建议包含：

- 页面转场动画。
- 设置二级菜单动画。
- 预测式返回动画。
- 卡片缩小消失返回动画。
- 图片切换渐变。
- 底部 Dock 切换反馈。
- 整理卡片滑动动画。
- 收藏 / 回收站操作反馈动画。

卡片返回动画不应只是固定滑走，更理想的效果是缩小到中心并逐渐消失。

### 震动反馈

可以加入：

- 底部 Dock 切换震动。
- 收藏成功轻震。
- 加入回收站轻震。
- 删除确认更明显的震动。
- 滑动达到阈值时轻震。

震动反馈应提供开关。

### 深色模式

深色模式要求：

- 背景统一。
- 文字可读。
- 卡片层级清楚。
- 不出现深色下文字不可见。
- 不出现奇怪白色渐变。
- 首页、设置、整理页、弹窗风格一致。

## 平板适配

平板适配目标：

- 首页卡片根据宽度自动重新排列。
- 整理页面不要只是居中一个手机宽度。
- 视频整理页适配宽屏。
- 图片预览利用更大空间。
- 设置页可以考虑双栏结构。
- 收藏 / 回收站可以多列网格。
- 手机端原布局不能被破坏。

推荐策略：

```text
Compact 宽度：手机布局
Medium 宽度：双列 / 较宽卡片
Expanded 宽度：三列 / 主从布局 / 大预览区
```

可使用：

- WindowSizeClass。
- BoxWithConstraints。
- LazyVerticalGrid。
- Adaptive grid。
- 自定义 breakpoints。

媒体删除：

- 删除前二次确认。

## 已知问题和重点验收项

图片整理相关：

- 点击放大后应能缩放查看细节。
- 用系统相册查看时应正确跳转到对应图片。
- 进入整理页时不应先显示“暂无队列”再显示图片。
- 图片切换时背景不应闪白。
- 手势判定不能过于模糊。
- 重新随机按钮不能破坏顶部布局。
- 回收站和收藏页应能放大查看图片。

视频整理相关：

- 视频整理页面不能闪退。
- 视频页面应适配平板宽度。
- 视频播放器生命周期必须正确释放。
- 快速切换视频不能崩溃。
- 视频加载失败需要兜底 UI。

设置相关：

- 设置页面应有清晰一级 / 二级菜单。
- 二级菜单跳转应有动画。
- 服务与支持应和设置内容统一。
- 可视化自定义应直观。
- 深色模式文字和背景对比必须可靠。

交互相关：

- 保留预测式返回效果。
- 返回动画不应过于固定。
- 卡片返回动画应自然。
- 底部 Dock 切换可带震动。
- 主要按钮视觉不能过薄。
- 深色模式背景统一。

## 开发硬约束

后续让 Codex 或其他 AI 编程工具继续改代码时，建议保留这些约束：

```text
1. SettingsScreen 不使用 @Composable Column.() -> Unit DSL trailing lambda。
   建议改用 List<@Composable () -> Unit> 传入设置项。

2. PhotoCleanScreen 中 AnimatedVisibility 如果出现隐式接收器冲突，
   使用全限定名 androidx.compose.animation.AnimatedVisibility。

3. MediaStoreScanner 中避免中文字符串模板 ${}，
   尽量使用显式字符串拼接，减少编译和编码问题。

4. Compose 1.8.2 避免使用 matchParentSize()。

5. Compose 1.8.2 避免使用 animateItemPlacement()。

6. 弃用图标优先替换为 Icons.AutoMirrored.*。

7. 不要加入会员、VIP、支付、广告、登录、云同步等功能。

8. 修改平板布局时不能破坏手机端原有排布。

9. 修改重新随机逻辑时不能影响顶部栏和当前页面结构。

10. 视频播放器必须处理生命周期释放，避免闪退。
```

项目命名应统一使用“看了么”/`kanleme`，新增包名、资源名、文档路径或持久化键时不要再引入旧产品命名。

## 非功能要求

性能要求：

- 媒体扫描不能阻塞主线程。
- 首页不能每次进入都全量扫描。
- 图片缩略图要缓存。
- 大图预览不能一次性加载超大原图导致卡顿。
- 视频切换时要及时释放资源。
- 平板端不能只是简单拉伸。
- 大量媒体情况下队列加载要稳定。

稳定性要求：

- 权限拒绝时不崩溃。
- 媒体文件不存在时不崩溃。
- URI 打不开时有提示。
- 视频加载失败时有兜底。
- 回收站恢复失败时有提示。
- 排除文件夹失效时可以移除。
- 设置项持久保存，不应重启丢失。

安全要求：

- 默认不永久删除。
- 加入回收站可恢复。
- 彻底删除必须确认。
- 不上传用户媒体。
- 不做云端处理。
- 不做账号体系。
- 不做广告化推荐。
